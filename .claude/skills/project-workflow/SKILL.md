Guides for day-to-day development on defi.institute. These resources define the non-negotiable rules for code style, testing, and releasing.

- [testing-rules.md](resources/testing-rules.md) — what must be tested and how
- [release-checklist.md](resources/release-checklist.md) — steps before deploying to production

## When to use this skill

Load coding-style.md before implementing any feature.
Load testing-rules.md when adding or reviewing tests.
Load release-checklist.md before any production deploy.
